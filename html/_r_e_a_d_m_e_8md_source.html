13 A Computational toolbox for large scale **Ca**lcium **Im**aging **An**alysis*
14 
15 
16 
17 
18 Recent advances in calcium imaging acquisition techniques are creating datasets of the order of Terabytes/week. Memory and computationally efficient algorithms are required to analyze in reasonable amount of time terabytes of data. This projects implements a set of essential methods required in the calcium imaging movies analysis pipeline. Fast and scalable algorithms are implemented for motion correction, movie manipulation and source and spike extraction.
19 
20 
21 ### Features
22 
23 * Handling of very large datasets
24 
25  * Memory mapping
26  * Frame-by-frame online processing (some functions)
27  * opencv-based efficient movie playing and resizing
28 
29 * Motion correction
30 
31  * Fast parallelizable open-cv and fft-based motion correction of large movies
32  * Run also in online mode (i.e. one frame at a time)
33  * non rigid motion correction
34 
35 * Source extraction
36 
37  * identification of pixles associated to each neuron/neuronal structure
38  * deals with heavily overlaping and neuroopil contaminated movies
39  * separates different sources based on Nonnegative Matrix Factorization algorithms
40 
41 * Denoising, deconvolution and spike extraction
42 
43  * spikes can be inferred from fluorescence traces
44  * also works in online mode (i.e. one sample at a time)

109 # Deconvolution and demixing of calcium imaging data
110 
111 The code implements simultaneous source extraction and spike inference from large scale calcium imaging movies. The code is suitable for the analysis of somatic imaging data. Implementation for the analysis of dendritic/axonal imaging data will be added in the near future. The following references provide the theoretical background and original code for the included methods.
112 
113 Pnevmatikakis, E.A., Soudry, D., Gao, Y., Machado, T., Merel, J., ... & Paninski, L. (2016). Simultaneous denoising, deconvolution, and demixing of calcium imaging data. Neuron 89(2):285-299, http://dx.doi.org/10.1016/j.neuron.2015.11.037. [Github repo](https://github.com/epnev/ca_source_extraction).
114 
115 Pnevmatikakis, E.A., Gao, Y., Soudry, D., Pfau, D., Lacefield, C., ... & Paninski, L. (2014). A structured matrix factorization framework for large scale calcium imaging data analysis. arXiv preprint arXiv:1409.2903. http://arxiv.org/abs/1409.2903.
116 
117 Friedrich J. and Paninski L. Fast active set methods for online spike inference from calcium imaging. NIPS, 29:1984-1992, 2016. [PDF](https://papers.nips.cc/paper/6505-fast-active-set-methods-for-online-spike-inference-from-calcium-imaging). [Github repository](https://github.com/j-friedrich/OASIS).

179 Test the system
180 ----------------
181 
182 **SINGLE PATCH**
183 
184 In case you used installation af point 1 above you will need to download the test files from
185 <https://github.com/agiovann/Constrained_NMF/releases/download/v0.3/Demo.zip>
186 
187 A. Go into the cloned folder, type `python demo.py`
188 
189 B. Using the Spyder (type `conda install spyder`) IDE.
190 
191  1. Unzip the file Demo.zip (you do not need this step if you installed dusing method 2 above, just enter the Constrained_NMF folder and you will find all the required files there).
192  2. Open the file demo.py with spyder
193  3. change the base_folder variable to point to the folder you just unzipped
194  3. Run the cells one by one inspecting the output
195  4. Remember to stop the cluster (last three lines of file). You can also stop it manually by typing in a terminal
196  'ipcluster stop'
197 
198 C. Using notebook.
199 
200  1. Unzip the file Demo.zip (you do not need this step if you installed dusing method 3 above, just enter the Constrained_NMF folder and you will find all the required files there).
201  2. type `ipython notebook`
202  3. open the notebook called demoCNMF.ipynb
203  4. change the base_folder variable to point to the folder you just unzipped
204  5. and run cell by cell inspecting the result
205  6. Remember to stop the cluster (last three lines of file). You can also stop it manually by typing in a terminal
206  'ipcluster stop'

209 **MULTI PATCH**
210 + Download the two demo movies [here](https://github.com/agiovann/Constrained_NMF/releases/download/v0.4-alpha/Patch_demo.zip) (courtesy of Dr. Sue Ann Koay from the Tank Lab, Princeton Neuroscience Institute, Princeton. NJ). Unzip the folder. Then in Spyder open the file demo_patches.py, and change the base_folder variable to point to the folder you just unzipped.
211 + Run one by one the cells (delimited by '#%%')
212 + Inspect the results. The demo will start a cluster and process pathes of the movie (more details [here](https://github.com/agiovann/Constrained_NMF/wiki/Processing-large-datasets)) in parallel (cse.map_reduce.run_CNMF_patches). Afterwards, it will merge the results back together and proceed to firstly merge potentially overlaping components (cse.merge_components) from different patches, secondly to update the spatial extent of the joined spatial components (cse.spatial.update_spatial_components), and finally denoising the traces (cse.temporal.update_temporal_components). THe final bit is used for visualization.

231 External Dependencies
232 ============
233 
234 For the constrained deconvolution method (```deconvolution.constrained_foopsi```) various solvers can be used, each of which requires some additional packages:
235 
236 1. ```'cvxpy'```: (default) For this option, the following packages are needed:
237  * [CVXOPT](http://cvxopt.org/) Required.
238  * [CVXPY](http://www.cvxpy.org/) Required.
239 2. ```'cvx'```: For this option, the following packages are needed:
240  * [CVXOPT](http://cvxopt.org/) Required.
241  * [PICOS](http://picos.zib.de/) Required.
242 
243 In general ```'cvxpy'``` can be faster, when using the 'ECOS' or 'SCS' sovlers, which are included with the CVXPY installation.
